# 存储技术知识库自动化工作流

## 🎯 目标
实现存储技术信息的自动收集、整理、生成Markdown文档，并同步到GitHub。

## 📁 目录结构

```
storage-knowledge-base/
├── 01-每日简报/          # 每日技术新闻和动态
```

## 🔄 自动化流程

### 阶段1: 信息收集 (每日执行)
1. **时间:** 每天早晨 8:00
2. **任务:**
   - 收集存储行业新闻
   - 监控云厂商博客更新
   - 抓取技术论坛讨论
   - 获取安全漏洞信息

### 阶段2: 内容处理 (自动)
1. **AI摘要生成:** 使用AI总结核心内容
2. **分类整理:** 按技术领域自动分类
3. **质量评估:** 评估内容的技术深度
4. **去重处理:** 避免重复内容

### 阶段3: 文档生成 (自动)
1. **模板填充:** 使用模板生成结构化文档
2. **格式优化:** 确保Markdown格式规范
3. **链接检查:** 验证所有链接有效性
4. **元数据添加:** 添加标签、分类等元数据

### 阶段4: GitHub同步 (夜间执行)
1. **时间:** 每天凌晨 2:00
2. **任务:**
   - 自动提交所有更改
   - 生成更新摘要
   - 推送到GitHub仓库
   - 生成同步报告

## ⚙️ 配置说明

### 1. 信息源配置
创建 `config/sources.yaml`:
```yaml
rss_feeds:
  - url: "https://aws.amazon.com/blogs/storage/feed/"
    category: "cloud_storage"
  - url: "https://cloud.google.com/blog/feeds/storage"
    category: "cloud_storage"
    
web_scrapers:
  - url: "https://www.storagereview.com/"
    selectors:
      articles: ".article-list"
      
apis:
  - name: "github_trending"
    endpoint: "https://api.github.com/search/repositories"
    params:
      q: "storage"
      sort: "stars"
```

### 2. 关键词配置
创建 `config/keywords.yaml`:
```yaml
primary_keywords:
  - "storage"
  - "cloud storage"
  - "AI storage"
  - "data management"
  
technical_terms:
  - "NVMe"
  - "Ceph"
  - "MinIO"
  - "object storage"
  - "block storage"
  - "file storage"
  
vendors:
  - "AWS"
  - "Azure"
  - "Google Cloud"
  - "阿里云"
  - "华为云"
```

### 3. GitHub配置
创建 `config/github.yaml`:
```yaml
repository: "https://github.com/yourusername/storage-knowledge-base"
branch: "main"
commit_message_template: "自动更新: {date} 的存储技术知识库更新"
sync_schedule: "0 2 * * *"  # 每天凌晨2点
```
