## 🔄 完整工作流程

### 每日执行流程
```
┌─────────────────────────────────────────────┐
│           每日早晨 8:00                     │
├─────────────────────────────────────────────┤
│ 1. 运行信息收集脚本                         │
│    python3 scripts/simple-collector.py      │
│    ↓                                        │
│    生成当日技术简报模板                     │
│    ↓                                        │
│    保存到 01-每日简报/YYYY-MM-DD-*.md      │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│           每日凌晨 2:00                     │
├─────────────────────────────────────────────┤
│ 2. 运行GitHub同步脚本                       │
│    ./scripts/sync-to-github.sh              │
│    ↓                                        │
│    提交所有更改到Git                        │
│    ↓                                        │
│    推送到GitHub仓库（如已配置）             │
│    ↓                                        │
│    生成同步报告                             │
└─────────────────────────────────────────────┘
```

## 🚀 立即开始的步骤

### 步骤1: 配置GitHub仓库
```bash
# 1. 在GitHub创建新仓库: storage-knowledge-base
# 2. 获取仓库URL: https://github.com/你的用户名/storage-knowledge-base.git

# 3. 配置本地仓库
cd ~/.openclaw/workspace/storage-knowledge-base
git remote add origin https://github.com/你的用户名/storage-knowledge-base.git
git branch -M main
```

### 步骤2: 设置GitHub访问令牌
```bash
# 1. 访问 https://github.com/settings/tokens
# 2. 生成新的Personal Access Token
# 3. 权限选择: repo (完全控制仓库)

# 4. 设置环境变量
echo 'export GITHUB_TOKEN="你的令牌"' >> ~/.bashrc
source ~/.bashrc
```

### 步骤3: 配置定时任务
```bash
# 编辑crontab
crontab -e

# 添加以下内容:
# 每天早晨8点收集信息
0 8 * * * cd /home/ubuntu/.openclaw/workspace/storage-knowledge-base && python3 scripts/simple-collector.py >> logs/collection.log 2>&1

# 每天凌晨2点同步到GitHub
0 2 * * * cd /home/ubuntu/.openclaw/workspace/storage-knowledge-base && ./scripts/sync-to-github.sh >> logs/sync.log 2>&1
```

### 步骤4: 测试完整流程
