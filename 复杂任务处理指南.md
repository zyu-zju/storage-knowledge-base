# 复杂任务处理指南

## 🎯 什么是复杂任务？

### 复杂任务的特征
1. **多维度分析** - 需要从多个角度考虑问题
2. **深度推理** - 需要逻辑推理和创造性思考
3. **策略规划** - 需要制定长期或复杂的计划
4. **不确定性处理** - 需要处理模糊或不完整信息
5. **技术深度** - 需要深入的技术理解和专业知识

### 存储技术领域的复杂任务示例
- 🔧 **架构设计** - 设计企业级存储解决方案
- ⚡ **性能优化** - 诊断和优化复杂性能问题
- 🔄 **数据迁移** - 规划大规模数据迁移策略
- 💰 **成本优化** - 设计多维度成本控制方案
- 🛡️ **安全加固** - 设计全面的安全防护体系
- 🌐 **多云策略** - 规划跨云平台的存储策略

## 🔧 复杂任务处理流程

### 1. 任务接收和分析
```bash
# 使用复杂度分析器
python3 scripts/simple-complex-handler.py "任务描述"

# 示例
python3 scripts/simple-complex-handler.py "设计企业级混合云存储架构方案"
```

### 2. 复杂度评估
系统自动评估任务复杂度：
- **低复杂度** (<4分) → 使用 `deepseek-chat`
- **中复杂度** (4-7分) → 使用 `deepseek-chat`
- **高复杂度** (7-10分) → 使用 `deepseek-reasoner`
- **非常高复杂度** (>10分) → 使用 `deepseek-reasoner`

### 3. 模型选择和处理
- **简单/中等任务** → 直接处理，快速响应
- **复杂任务** → 使用reasoner进行深度分析
- **非常复杂任务** → 分阶段深度处理，生成详细报告

### 4. 结果生成和整合
- 生成详细的技术分析报告
- 创建知识库条目
- 保存所有相关文件
- 更新知识库索引

## 📁 文件组织结构

### 任务文件夹结构
```
复杂任务处理/
└── YYYYMMDD-HHMMSS/          # 任务ID（时间戳）
    ├── task-info.json        # 任务基本信息
    ├── prompt.txt            # 给reasoner的提示词
    ├── analysis-result.md    # 深度分析结果
    ├── 知识库条目.md         # 知识库整合条目
    └── 相关资源/             # 附加资源（可选）
```

### 文件说明
1. **task-info.json** - 任务元数据，包括复杂度分析、时间戳等
2. **prompt.txt** - 生成给AI模型的详细提示词
3. **analysis-result.md** - 完整的深度分析报告
4. **知识库条目.md** - 简化的知识库条目，便于检索
